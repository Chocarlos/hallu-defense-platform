# Code of Conduct

## Purpose

Hallu Defense Platform welcomes technical discussion, testing, documentation, review, and responsible security research. Participation must remain professional, evidence-based, and safe for contributors and users.

## Expected conduct

Participants are expected to:

- communicate respectfully and focus criticism on ideas, code, evidence, and outcomes;
- state uncertainty, assumptions, conflicts of interest, and validation limits honestly;
- protect secrets, personal data, tenant data, unpublished vulnerabilities, and private communications;
- respect licensing, authorship, attribution, and the right of contributors to decline participation;
- accept maintainer decisions on scope, risk, release readiness, and repository safety;
- help correct mistakes without harassment, humiliation, threats, or retaliation.

## Unacceptable conduct

The following behavior is not permitted in repository spaces or project-related interactions:

- harassment, discrimination, intimidation, stalking, threats, or sustained personal attacks;
- publishing another person's private information without explicit permission;
- sharing credentials, exploit details, tenant data, or sensitive logs in public issues or pull requests;
- knowingly fabricating test results, security evidence, benchmarks, provenance, or compatibility claims;
- submitting malicious code, hidden telemetry, backdoors, destructive payloads, or intentionally misleading changes;
- spam, coordinated disruption, impersonation, or attempts to bypass repository security controls.

## Security reports

Do not include vulnerability details, proof-of-concept exploits, secrets, private logs, or tenant data in public discussions. Follow [`SECURITY.md`](SECURITY.md) and use the repository's private reporting channel when it is available.

## Enforcement

Maintainers may edit or remove content, close discussions, reject contributions, restrict participation, or report abuse to GitHub when conduct threatens people, the project, or its users.

Enforcement should be proportional and documented privately where appropriate. Repeated or severe violations may result in permanent exclusion. Good-faith disagreement, respectful review, and evidence-based criticism are not violations.

## Reporting conduct concerns

Report conduct concerns privately to the repository owner, `@Chocarlos`, through an available private contact channel. When no private channel is available, open a neutral issue requesting private contact without publishing incident details.
